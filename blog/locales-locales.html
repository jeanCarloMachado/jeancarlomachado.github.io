Locales, locales...

September 2, 2015

This is a very tiny post, it’s for me to remember about locales. Don’t matter which kind of Backend app one develop, if it will run in more than one machine you MUST set the locale. I know, I know, how can any developer install an SO that is not in English?! Don’t ask me, put the setting.

And this is the way of doing it:

ini_set('intl.default_locale', 'en-US');

I decided to place it here to be able to avoid searching in the project history for the notation when (not to far) it will be needed once again.
